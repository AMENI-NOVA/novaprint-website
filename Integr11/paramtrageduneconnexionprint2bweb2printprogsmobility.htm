Paramétrage d’une connexion Print2B/Web2Print Pro/GsMobility

Précédent   Suivant

Pour paramétrer une nouvelle connexion, sélectionnez « Connexions », puis cliquez  sur Nouveau dans le menu Paramétrages ou contextuel.

Types de connexions Print2B/Web2Print Pro/GsMobility

Il existe deux types de connexions Print2B/Web2Print Pro/GsMobility :

¨   les connexions « Client » : les clients n’auront accès qu’à leurs devis et auront des fonctions limitées.

¨   les connexions « Interne » pour les employés (commerciaux) :

·    Print2B : les employés auront accès à l’application comme si elle était accessible en local.

·    GsMobility : pour créer une connexion utilisable depuis GsMobility

Pour chaque connexion, interne ou externe, vous devez créer une entrée dans la rubrique « Connexions ».

L’écran de paramétrage de la connexion est adapté en fonction du type de connexion : client ou interne.

Dans les 2 cas, vous pouvez indiquer un préfixe à utiliser pour numéroter les devis créés depuis cette connexion.

Correspondance entre les connexions Print2B Windows et les comptes d'utilisateurs du serveur

Lorsque le serveur d’applications est également utilisé en interne (ou si le serveur Web est aussi le serveur d’applications interne), au démarrage de l’application, Graphisoft cherche s’il existe une connexion Print2B ayant le même nom que le nom d'ouverture de session de l'utilisateur sur le serveur.

Le comportement de l'application au démarrage dépend des paramètres de la connexion Print2B correspondant au nom de session de l'utilisateur :

¨   La connexion Print2B est de type "Client" :

L'application démarre avec l'interface Print2B simplifiée.

¨   La connexion Print2B est de type "Interne" :

·    Le choix indiqué pour le paramètre "A qui est destiné cette connexion ?" est "Tous" (Print2B uniquement) :

L'utilisateur devra alors s'identifier parmi les utilisateurs enregistrés (employés).

·    Le choix indiqué pour le paramètre "A qui est destiné cette connexion ?" est un des utilisateurs de la liste :

L'application démarrera en utilisant cet utilisateur sans qu'il soit nécessaire de s'identifier.

¨   Aucune connexion Print2B ne correspond au nom d'ouverture de session utilisé :

La fenêtre d'identification apparaît et permet à l’utilisateur de s’identifier.

Connexion Print2B/Web2Print Pro Client

Paramètres d’identification client

Un compte Print2B/Web2Print Pro est associé à un interlocuteur d’une société.

On attribue à chaque compte un nom de connexion (login) et un mot de passe.

Le mot de passe n’est utilisé que par Web2Print Pro.

Dans le cas de Print2B, l’identification est réalisée par la session Windows. Lorsque Graphisoft démarre, l’interface utilisateur Print2B s’active automatiquement si le nom de session Windows correspond à une connexion Print2B de type « Client ».

Paramètres de la connexion

¨   Préfixe pour le n° de devis : les devis créés par cet utilisateur seront préfixe par la valeur indiquée ici

¨   Modèle de lettre de confirmation : modèle d’édition pour l’affichage de l’offre de prix.

Cette fonctionnalité est toujours accessible.

Le modèle d’édition sélectionné doit répondre à 2 contraintes :

·    Il ne doit être liée à aucune imprimante.

·    Il doit correspondre à la langue indiquée pour la société à laquelle se rapporte la connexion Print2B.

Pour plus d’informations reportez-vous au paragraphe « Paramétrage des modèles d’édition pour les connexions Print2B » plus loin dans ce chapitre.

¨   Dossier technique : modèle d’édition pour l’affichage du dossier technique.

Pour empêcher l’accès au dossier technique, sélectionnez « Non autorisé » dans la liste.

¨   Devis des autres interlocuteurs : autorise ou non un utilisateur Print2B à voir les devis faits par d’autres interlocuteurs de sa société.
NB : cette option ne concerne pas les connexions GsMobility.

¨   Modèle de devis

Indique si le client à accès aux modèles de devis déjà enregistrés pour créer de nouveaux devis.
Les choix possibles sont :

·    Non autorisé : le client ne peut pas utiliser de modèle pour saisir de nouveaux devis.

·    Seulement les modèles du client : seuls les modèles de devis enregistrés pour ce client sont disponibles.

·    Modèles du client et sans client : le client à accès aux modèles de devis enregistrés pour lui, mais aussi aux modèles qui n'ont pas de client.

¨   Calcul des devis : autorisé ou non.

Si le calcul n’est pas autorisé, le client ne pourra qu’enregistrer ses devis en attente. Le fabricant en sera alors averti (voir le paragraphe « Notifications à l’imprimeur » plus loin dans ce chapitre).

¨   Tableau de charge :

Indique si le client peut accéder au tableau de charge. Le tableau de charge contient les commandes correspondant aux devis (fait par le client en PrintToB) validés par l'imprimeur.

NB : l’accès au tableau de charge n’est pas disponible avec Web2Print Pro

¨   Gestion des articles

Indique si le client a accès aux fonctionnalités liées à la gestion des articles : liste des articles, détail d’un article, devis ou commande à partir d’un article.

NB : la gestion des articles n’est disponible qu’avec Web2Print Pro.

¨   Création de commande

Indique si les commandes crées sont des commandes prévionnelles ou des commandes fermes.

Les commandes prévisionnelles doivent être validées dans Graphisoft pour apparaître dans la GPAO.

¨   Téléchargement des fichiers

Autorise l’utilisateur à télécharger des fichiers pour les commandes sur le serveur de l’imprimerie.

¨   Etat d’avancement des commandes

Lorsque l’état d’avancement des commandes est détaillé, la fenêtre de détail d’une commande affiche cet état d’avancement sous la forme de cadres de couleurs pour le prépresse, l’impression, le façonnage et la livraison.

¨   Accès au portail web du prépresse : Indique si le client a accès au portail web du flux prépresse.

Si l’accès est autorisé, il faut cliquer sur « Détails » pour saisir l’identifiant et le mot de passe de ce client dans le portail du prépresse.

Pour Agfa Portal, la saisie de l’identifiant et du mot de passe n’est pas nécessaire.

Pour Kodak InSite, la saisie de l’identifiant et du mot de passe est requise.

Si l’accès est autorisé, une colonne « Portail Prépresse » est affichée dans la liste des commandes de WebToPrint Pro, ainsi qu’un bouton « Portail Prépresse » dans le détail d’une commande.

 

Connexion Print2B interne ou GsMobility

A qui est destiné cette connexion

¨   Si vous sélectionnez, « Tous », tous les employés pourront utiliser l’application en Print2B et devront s’identifier avant de pouvoir rentrer dans l’application.
Ce choix n’est pas disponible si vous disposez de droits GsMobility

¨   Sélection d’un employé :

·    En mode Print2B, c’est cet utilisateur qui sera automatiquement sélectionné lors de l’entrée dans le programme.

·    Avec GsMobility, ce choix définit l’employé associé à la connexion.
