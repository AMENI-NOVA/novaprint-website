Dans l’écran de paramétrage, "Paramétrages généraux – Gestion des en-cours clients",

on peut décider de gérer l’en-cours :

¨   pour tous les clients

¨   seulement pour les clients avec un en-cours maximum défini

¨   pour aucun client.

Définissez le mode de gestion que vous souhaitez : automatique ou non.

Si vous optez pour la gestion automatique,  on y définira le nombre de jours après la date d’échéance pour considérer qu’un effet est payé. Exemple : Echéance le 15/10, délai 5jrs.  Si on est le 17/10, l’effet est encore considéré dans l’en-cours client. Le 21 il n’y sera plus. C’est le délai nécessaire pour que la banque encaisse l’effet et prévienne en cas de non-paiement.

Avec l’option gestion manuelle, l’effet n’est payé que si vous le marquez « Réglé ». Si vous avez une interface comptable comptatible définie pour l’export des factures de vente, le bouton « Import des règlements » est actif et vous pouvez paramétrer cette fonctionnalité en fonction de votre comptabilité.

Remarque : L’effet reste dans la liste 730 jours après qu’il soit considéré payé. Ensuite il est effacé.

Enfin, on peut indiquer le taux de TVA par défaut auquel on veut que soient taxées les commandes qui entrent dans les en-cours.

Le paramétrage des en-cours clients n'est visible que pour les utilisateurs qui disposent du droit "Accès au paramétrage des encours clients".

Remarque : Les effets créés manuellement et les modifications d’effets seront supprimés si vous changez le paramétrage de gestion des en-cours. En effet, lorsque vous changez ce paramétrage, la liste des effets à prendre en compte dans les en-cours est intégralement reconstruite à partir des commandes, factures et relevés.

 

Pour chaque employé administratif (paramétrage des droits aux fonctionnalités), on définira alors les droits suivants.

¨   Droit de modifier les paramètres de gestion des en-cours clients.

¨   Droit de voir les en-cours client.

¨   Droit d’intervenir sur les effets (échéance, échelonnement).

¨   Droit d’accepter un dépassement d’en-cours pour créer une commande.
