Interface avec le prépresse

Précédent   Suivant

L'interface JDF avec le prépresse, permet à Graphiplus d'envoyer toutes les informations sur la structure d'un devis : imposition, nombre d'exemplaires, de pages, d'éléments, de couleurs, …

Ces informations sont transmises, soit automatiquement, lors de la transformation d'un devis en commande, soit manuellement depuis la liste des commandes.

 

Si votre "système de prépresse" le permet, cette interface permet aussi de récupérer dans Graphifab les informations suivantes pour chaque dossier traité :

¨   Nombre de plaques produites,

¨   Nombre de plaques refaites facturables au client,

¨   Nombre de plaques refaites non facturables,

¨   Nombre de plaques validées ou refusées,

¨   Nombre d'épreuve de pages,

¨   Nombre d'épreuve d'imposition.

¨   Aperçu du contenu des pages du dossier (Agfa Apogee)

¨   Pourcentage des pages reçues (Agfa Portal et Kodak InSite)

 

Voir le détail des données échangées la partie Annexe de ce document.

 

Si vous gérez les plaques dans la gestion de stock de Graphisoft, celles-ci sont automatiquement déduites du stock.

 

L'interface avec le prépresse répond aux normes JDF suivantes, définies par le CIP4 :

¨   JDF version 1.2 et 1.3

¨   Base Conformance Levels ICS version 0.5-v14 (Base_L1-1.0)

¨   MIS base Conformance ICS version beta 0.5 niveau 1 (MISBC_L1-1.0)

¨   MIS to Prepress ICS version beta 0.5 niveau 1 (MISPRE_L1-1.0)
