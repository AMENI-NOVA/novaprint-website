Bons de commande

Précédent   Suivant

Un bon de commande se compose de :

¨   Une entête

¨   Une ou plusieurs commandes

Champs de l’entête du bon de commande

¨   Imprimerie : vos coordonnées

¨   Fournisseur : Nom, adresse, interlocuteur, téléphone, fax, conditions de paiement, informations administratives et techniques,

¨   Document : Nom, date, répétition, n° de pages. Cf. lettre de confirmation.

¨   N° de bon de cme

¨   Référence

¨   Référence externe

¨   Date d’envoi prévue globale (vide si inconnue)

¨   Date de livraison (retour) prévue globale (vide si inconnue)

¨   Montant HT commande (vide si égal à 0) : somme des montants des commandes se trouvant dans le bon.

¨   Suivi par : personne de votre entreprise qui suit ce bon de commande.

 

Adresse de livraison :

Les champs suivants sont initialisés avec l’adresse de la société émettrice (société utilisatrice) du bon de commande.

En Optiplus, cette adresse correspond à l’entrepôt défini dans le stock du la première ligne du bon de commande. Si cette première ligne n’est pas liée à un stock, l’adresse est vide.

¨   Raison sociale livraison

¨   Adresse de livraison complète

¨   Ligne d’adresse livraison

¨   Code postal livraison

¨   Ville livraison

 

Champs d’une commande

Les champs décrivant les commandes doivent être inclus dans un bloc de répétition, pour que toutes les commandes du bon soient éditées.

Ces champs sont les suivants :

¨   N° du dossier concerné par la commande (un bon de commande peut regrouper des commandes concernant des dossiers différents).

¨   Désignation référence (ou Dossier - Descriptif)

¨   Quantité commandée

¨   Prix unitaire

¨   Unité utilisée pour le prix unitaire

¨   Montant HT référence : montant HT de la commande

¨   Date d’envoi prévue référence

¨   Date d’envoi réelle référence

¨   Date de livraison prévue référence

¨   Délai de livraison référence

¨   Référence fournisseur : pour les commandes d’articles gérés en stock

¨   Stockage : position de stockage : pour les commandes d’articles gérés en stock

¨   Dossier de fabrication construit à partir d’un modèle préétabli.
(Attention: l’inclusion d’un dossier en tableau dans la cellule d’un tableau du bon de commande n’est pas possible et provoquera une erreur).

¨   N° Ligne : numéro de la ligne de commande dans le bon de commande.

¨   Dossier : regroupe les champs provenant du dossier concerné par la ligne du bon de commande

¨   Stock : regroupe les champs provenant de la commande de stock concernée par la ligne du bon de commande.
