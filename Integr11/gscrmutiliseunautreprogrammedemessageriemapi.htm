GS CRM utilise un autre programme de messagerie (Mapi)

Précédent   Suivant

Pour envoyer et recevoir des e-mails en utilisant un programme de messagerie :

¨   Vous devez indiquer "Utiliser le programme de messagerie installé sur cet ordinateur" dans les préférences de la messagerie.

¨   Vous devez avoir installé sur votre ordinateur un utilitaire de messagerie (compatible Mapi) comme « Outlook express », « Netscape Messenger », « Eudora » ou autre.

¨   Votre accès à une boîte aux lettres électronique doit être correctement paramétré.
A titre de vérification, vous devez pouvoir exécuter l’utilitaire de messagerie et vous envoyer un message à vous-même.

¨   Dans les paramètres de votre utilitaire de messagerie vous devez avoir indiqué qu’il est un client MAPI. Ceci permet à d’autres applications, comme Graphisoft, de l’utiliser pour l’envoi de courrier électronique. En fonction des configurations cette étape n'est pas toujours nécessaire.

¨   Pour les postes fonctionnant sous Windows 95, il peut être nécessaire d’installer le « Gestionnaire de messagerie » du groupe « Communication » de l’installation de Windows 95. Contactez votre installateur si nécessaire pour procéder à cette installation.

 

Lors de la lecture des e-mails reçus, GS CRM va lire la boîte de réception "principale" de votre messagerie. Si vous souhaitez récupérer d’autre compte de messagerie, ceux-ci doivent « arriver » aussi dans la boite de réception « principale » ou « par défaut » de la messagerie.

 

Si vous avez plusieurs comptes de messagerie (plusieurs adresses e-mails) à récupérer dans GS CRM, plusieurs méthodes sont envisageables :

¨   Un ordinateur par compte à récupérer : Chaque compte doit être paramétré dans le programme de messagerie sur un ordinateur différent. Ainsi, chaque ordinateur aura dans sa messagerie Internet les messages destinés à une seule adresse e-mail. Sur chacun des ordinateurs concernés, il faut activer la récupération des messages par GS CRM.

¨   Un seul ordinateur pour tous les comptes de messagerie : Tous les comptes sont paramétrés sur un seul ordinateur. La récupération des messages par GS CRM ne doit être activée que sur cet ordinateur.

¨   Ou un mélange des deux méthodes.

 

GS CRM et Microsoft Exchange

Si vous utilisez un serveur de messagerie Microsoft Exchange, il peut arriver que les destinataires des e-mails, qui n’ont pas une messagerie Microsoft, reçoivent un fichier WinMail.dat au lieu de vos pièces jointes.

Pour corriger cela Microsoft a publié l’article suivant : http://support.microsoft.com/kb/841668

 

GS CRM et Thunderbird

Pour fonctionner correctement, il faut s’assurer qu’un compte par défaut est bien défini dans Thunderbird.

Si ce n’est pas le cas une erreur 32053 peut s’afficher lors de l’envoi des e-mails.

 

Même s’il y a un seul compte de messagerie de défini, il n’est pas obligatoirement par défaut.

Voici la démarche avec la version 3.0.3 de Thunderbird :

¨   Ouvrir le menu "Outils" puis "Paramètres des comptes" ;

¨   Cliquer sur le compte de messagerie (PAS sur "Dossiers locaux" !) dans l'arbre de gauche pour le sélectionner ;

¨   Ouvrir le menu/liste "Gestion des comptes" en bas de l'arbre ;

¨   Cliquer sur "Compte par défaut"
