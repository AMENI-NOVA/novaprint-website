Paramétrage des connexions JDF

Précédent   Suivant

Pour chaque système JDF (flux de prépresse, presses, …) devant communiquer avec Graphisoft, vous devez créer une connexion JDF.

La connexion permet de définir avec qui on travail, quelles sont les informations échangées et comment sont fait les échanges.

 

La liste des connexions est disponible depuis la feuille de paramétrage de Graphiplus, dans JDF (Job Definition Format), puis Connexions JDF.

 

Contenu du JDF

Cet onglet défini le contenu du fichier JDF

A qui envoyer le JDF ?

Choisissez le nom du partenaire JDF avec qui Graphisoft doit communiquer.

Si vous ne trouvez pas le nom de votre flux de prépresse ou de votre constructeur de presse, choisissez « Autres ».

Version du JDF

Indiquez ici la version du JDF accepté par le partenaire.

Que doit contenir le JDF ?

Indiquer ici ce que doit contenir le fichier JDF envoyé.

Si vous cochez « Le prépresse », le fichier JDF contiendra toutes les informations concernant le flux de prépresse.

« Envoyer l’imposition dans le JDF » : en fonction de la configuration du flux de prépresse, décochez cette case s’il n’accepte pas de recevoir l’imposition.

« Demander les épreuves de pages » : Si cette case est cochée, des épreuves de pages seront demandé systématiquement pour chaque travaux envoyés au flux de prépresse.

« Taille des fonds perdus » : Dans les données technique, la zone « Fond perdu entre deux poses » permet de définir ce qui sera effectivement rogné entre deux poses. Ici « Taille des fonds perdus » représente la valeur du fond perdu tout autour de la pose. Cette valeur est comprise dans la rogne.

 

Si vous cochez « L’impression », les informations d’impression seront envoyées.

Sélectionnez ensuite les presses JDF concernées par cette connexion.

La liste présente tous les postes de gestion de production interfacés en JDF et de type « Impression conventionnelle ».

Si vous cochez « Envoyer un fichier JDF par fiche de travail d’impression », il y aura autant de fichiers JDF envoyés que de fiches de travail d’impression. Si cette case n’est pas cochée, un seul fichier est envoyé contenant toutes les donnés d’impression.

 

Cochez « Envoyer SEULEMENT le fichier JDF s’il y a de l’impression » pour les connexions correspondant à des presses. Cette option évite d’envoyer à une presse un dossier sans traitement d’impression.

 

« Regrouper les intérieurs d’une brochure » : Dans le cas d’une brochure qui contient plusieurs intérieurs (avec des formats, couleurs ou papier différents). Si cette option est cochée, un seul et unique travail pour l’intérieur sera envoyé au prépresse.

Il est recommandé de cocher cette option, mais certain partenaire JDF ne supporte pas cette organisation.

 

Connexion

Cet onglet défini comment est envoyé le JDF

Où faut-il envoyer le JDF ?

 

Méthode d’envoi

¨   Enregistrer le fichier JDF dans le répertoire : Les informations JDF sont enregistrées dans le répertoire défini dans "Dossier d'envoi". Le partenaire JDF se charge de vérifier régulièrement si de nouveaux fichiers sont arrivés.

¨   JMF SubmitQueueEntry et JDF dans le répertoire : Un message JMF de type SubmitQueueEntry est envoyé à la presse. Il indique qu'un nouveau dossier à été enregistré dans le répertoire défini dans "Dossier d'envoi".

¨   JMF SubmitQueueEntry et JDF incorporé : Un message JMF de type SubmitQueueEntry est envoyé. Le dossier est envoyé en même temps que le message et n'est pas enregistré dans un répertoire.

Dossier d'envoi

Il faut indiquer ici le répertoire de stockage des fichiers JDF. Utilisez le bouton <…> pour afficher un écran de sélection de répertoire.

Cette saisie n'est pas possible si la méthode d'envoi est JMF SubmitQueueEntry et JDF incorporé.

Adresse http

Si la méthode d'envoi utilise le message JMF SubmitQueueEntry, il faut indiquer ici l'adresse http du programme de prépresse.

Utilisateur & mot de passe

Si le partenaire JDF le demande, saisissez un nom et un  mot de passe.

Tester la connexion

Ce bouton ouvre une fenêtre permettant de vérifier la communication http entre Graphisoft et le partenaire JDF.

Pour fonctionner, une adresse http doit être saisie.

Envoi & Abonnements

Cet onglet défini quand est envoyé le JDF.

 

Quand faut-il envoyer le JDF ?

¨   A la demande :
Cette option fait apparaître la fonction "Exporter la commande en JDF" dans la liste des commandes.

¨   Lors de la transformation d'un devis en commande :
Cochez cette option pour envoyer automatiquement la commande en JDF dès sa création.
Si l'option suivante est cochée, seules les fiches prêtes sont envoyées lors de la création de la commande.

¨   Dès qu'une fiche est prête à être traitée :
Avec cette option, les fiches de travail ne sont envoyées aux machines JDF qu'à partir du moment où elles deviennent réalisables.
Ainsi, une presse ne peut pas recevoir des informations sur un travail où les plaques ne sont pas commencées. Le terminal de la presse ne propose que les travaux "possibles".
Cette option n'a d'intérêt que si vous avez l'Intégrale. Sans le temps réel, les fiches de travail sont systématiquement prêtes.

¨   Renvoyer si la commande ou si une fiche est modifiée :
Dès qu'une fiche (ou une commande) est modifiée, celle-ci est renvoyée. Les modifications sont les suivantes :

·    Modifications des temps restants,

·    Transfert de la fiche d'un poste à un autre,

·    Modification de la planification,

·    Modification des propriétés de la commande.

 

Quand s’abonner ?

L’abonnement à un partenaire JDF permet d’être informé régulièrement de son état et de ce qu’il fait.

¨   Ne pas s’abonner : Aucun demande d’abonnement ne sera envoyée ;

¨   S’abonner au démarrage de ServeurJMF : lors du lancement « ServeurJMF », le programme envoi une demande d’abonnement à cette connexion JDF. En s’abonnant ainsi, vous serez sera informé de tous que fait le partenaire JDF, même pour des dossiers non envoyés par directement Graphisoft.

¨   S’abonner lors de l’envoi du JDF : La demande d’abonnement est faite lors de l’envoi du JDF. Cette demande est spécifiquement pour le dossier envoyé.

Trames

Cet onglet permet définir les trames utilisées en fonction du type de papier.

Cette saisie est active seulement si la case « Prépresse » est cochée dans l’onglet « Contenu du JDF ».

Quelles trames utilisez-vous ?

Ce premier tableau permet de définir les trames que vous souhaitez utiliser

Nom de la trame : Ce nom sert identifier la trame dans le tableau de correspondance avec le papier. Ce nom n’est pas envoyé dans le JDF.

Famille de trame : Nom de la famille de la trame.

Linéature : Linéature de la trame.

Forme du point : Sélectionnez dans la liste la forme du point de la trame.

 

Quand les trames sont-elles utilisées ?

Ce tableau permet de définir les correspondances entre les grades de papier (couché mat, couché brillant, non couché, …) et les trames à utiliser.

La colonne « Type et Famille » contient tout d’abord les 5 grades de papier normalisé (ISO 12647-2), puis le nom des catégories de vos papier.

Le principe est le suivant :

1. Dans le paramétrage des catégories de papier, vous renseignez au préalable le grade du papier.

2. Pour chaque grade de papier vous renseignez ici la trame à utilisé fonction du grammage.

3. Si une catégorie de papier utilise une trame spécifique (différente de celle associée à son grade), ajoutez cette catégorie dans le tableau

 

Rappel : Fonctionnement des tableaux :

Ajouter une ligne : Saisissez les informations sur la ligne indiquée par une étoile.

Modifier une ligne : Cliquez sur la colonne concernée et saisissez la modification.

Supprimer une ligne :Sélectionnez la ligne en cliquant sur la première colonne grise à gauche. Un triangle noir indique que la ligne est sélectionnée. Ensuite appuyez sur la touche « Suppr. ».

 

Plaques

Cet onglet permet de définir le format de plaques utilisées.

Pour chaque presse définie dans le paramétrage du devis, indiquez ici le format des plaques utilisées.

Cela permet au flux de sélectionner le CTP le plus adapté et les bonnes plaques.

Actions dans le plan

Cet onglet n’est visible que si le flux de prépresse « Agfa » est sélectionné.

Cet onglet permet de définir les actions qui seront ajoutées automatiquement dans le plan d’Apogee.

Les actions ne sont pas directement modifiables dans cet écran. Il faut passer par un fichier au format xml contenant les actions. Ce fichier vous sera fourni par AGFA en fonction de votre configuration et de vous souhaits.

Vous pouvez ici :

·      Ouvrir un fichier d’actions ;

·      Enregistrer les actions dans un fichier ;

·      Supprimer les actions envoyées via le JDF ;

·      Utiliser les actions par défaut (elles sont fournies à titre d’exemples).

Pour modifier les actions de la connexion JDF, il faut :

1.  Enregistrer les actions dans un fichier

2.  Modifier ce fichier avec un éditeur de texte ou xml (ex : NotePad++)

3.  Ouvrir le fichier des actions dans la connexion JDF.

Device ID

Cet onglet permet de définir plusieurs identifiants de machine (Device ID) pour une même machine en fonction du partenaire JDF et en fonction du type de travail (recto seul, basculé, retiration, …).

Cela permet aussi de "parler" d'une même machine à plusieurs partenaires JDF où celle-ci ne serait pas connue sous le même nom.

 

Le tableau présente tous les postes de gestion de production interfacés en JDF. Chaque poste est présenté sur deux lignes :

·      La première ligne indique l’identifiant JDF et le nom du poste dans la gestion de production.

·      La seconde  permet de saisir l’identifiant JDF à utilisé en fonction du type de travail. Par défaut, il est identique à celui saisi dans le paramétrage du poste.

Si vous avez utilisé la fonction « Tester la connexion », une liste vous propose les identifiants connus par cette connexion JDF.
