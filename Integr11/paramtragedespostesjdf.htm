Paramétrage des postes JDF

Précédent   Suivant

Pour indiquer qu'une machine de votre atelier est interfacée en JDF, il faut aller dans le paramétrage du poste correspondant dans la gestion de production.

Attention : le poste doit être détaillé par opération pour que l'interface fonctionne correctement.

 

Le bouton <Interfaces JDF> permet d'afficher le "paramétrage JDF" du poste.

 

Type de l'interface

Indique le type de l'interface utilisée pour ce poste. Trois choix possibles:

¨   Poste non interfacé : Le poste n'a pas d'interface JDF.

¨   Impression conventionnelle : Ce poste est une presse qui est interfacée en JDF.

¨   Prépresse : Ce poste correspond à une machine ou logiciel de votre système de prépresse.
Les zones "Poste" et "Type d'opérations du poste" ne sont pas utiles pour cette interface.

 

Identifiant JDF

Cette zone sert à saisir l'identifiant de la machine au sein de votre "parc JDF". Deux machines ne doivent pas avoir le même identifiant JDF au sein de votre imprimerie.

FriendlyName

Indiquez ici le nom usuel de la machine. Cette information est optionnelle. Certain partenaire JDF la requière afin d’identifier exactement la machine à sélectionner.

Type de presse

Seulement si le type d'interface est "Impression conventionnelle".

Indiquez ici le type de votre presse : presse feuille ou presse bobine.

Créer une fiche de travail par passage...

Seulement si le type d'interface est "Impression conventionnelle".

Si cette case est cochée, lors de la création de la commande, il sera créé autant de fiche de travail d’impression que de passage en machine.

Exemple : Pour un tirage « côté par côté » il y à deux passages prévus sur la machine : un pour le recto, l’autre pour le verso. Si cette case est cochée, deux fiches de travail seront créées : une pour le recto, une pour le verso.

·      Cela va permettre à la machine interfacée en JDF de recevoir autant de travail d’impression que de passage. L’opérateur peut choisir passage qu’il souhaite traité en premier ;

·      Les temps retournés seront associés au passage correspondant ;

·      Les quantités affichées sur le pupitre de la presse correspondent bien au passage à effectuer et non à la globalité du travail.

Utiliser les messages JMF pour remplacer un terminal atelier

Cochée cette option si vous souhaitez utiliser les messages JMF pour renseigner automatiquement les temps et opérations de la gestion de production.

Attention  Cette option consomme un droit atelier.

Graphiplus vers JDF

Seulement si le type d'interface est "Impression conventionnelle".

Ce tableau permet de typer les opérations du poste. Cela permet d'envoyer à la presse des informations détaillée sur le temps de calage, de roulage et de lavage.

A gauche sont affichées les opérations définies sur ce poste et à droite les types d'opérations connus en JDF.

Pour chaque opération, sélectionnez le type JDF le plus approprié.

JDF vers Graphiplus

Ce tableau permet de définir le lien entre les informations envoyées par une machine JDF et les opérations du poste de gestion de production.

A gauche sont affichées les opérations définies sur ce poste et à droite les types d'informations envoyées par le JDF.

Status JMF :

Pour chaque opération, sélectionnez le type d'information le plus approprié.

 

Coût des opérations :

Cette colonne n’est utile que pour les postes « prépresse » comme le CTP ou l’épreuve d’imposition.

Le prépresse n’envoi aucune information de temps mais seulement un nombre d’opérations réalisées. Le fait de définir ici un coût d’opération, permet à Graphisoft de calculer le temps de l’opération en fonction du prix de l’heure du poste.

Les presses envoyant les informations de temps, il n’est pas utile de saisir un coût d’opération. Il sera déduit automatiquement en fonction du prix de l’heure du poste et du temps indiqué.

 

Condition :

Cette colonne permet de restreindre l’utilisation de cette opération à une condition donnée.

Les conditions ne s’appliquent que pour les status JMF suivants :

·      Création plaque

·      Epreuve imposition

 

Par exemple, il est possible de créer une opération pour les plaques 1030x800 (avec une condition sur la dimension) et une autre pour les plaques 745x605. Chaque type de plaque peut ainsi avoir un cout différent. Lorsqu’une plaque est créée, le programme choisi l’opération ayant pour condition le format ce celle-ci.

Si aucune opération ne correspond, le message JMF correspondant est affiché en erreur dans GSTraiteHttp.

 

 

Liste des conditions :

·      Pas de conditions : Aucune condition n’est appliquée pour choisir cette opération

·      Media@Brand : La condition s’applique sur l’attribut Brand du nœud Media correspondant à la plaque ou à l’épreuve.

·      Media@Dimension : La condition s’applique sur l’attribut Dimension du nœud Media correspondant à la plaque ou à l’épreuve.

·      Media@UserMediaType : La condition s’applique sur l’attribut UserMediaType du nœud Media correspondant à la plaque ou à l’épreuve.

 

Valeur dans le JMF :

La condition est remplit si la valeur saisie ici est identique à celle dans le JMF. Identique à la décimale prêt.

C’est pourquoi la dimension (condition Media@Dimension) doit être saisie comme dans le JMF, soit en point pica (72ieme de pouce). La valeur en millimètre est affichée à titre indicatif.

 

Comment connaitre les valeurs des messages JMF ?

Pour connaitre avec exactitude les valeurs à indiquer dans la condition, voici comment il est possible de procéder :

En bref :

Le principe consiste à définir des conditions qui seront systématiquement fausses. Ainsi les messages JMF correspondant seront affichés en erreur dans GSTraiteHttp.

En ouvrant ces messages, on peut trouver les valeurs exactes à mettre dans les conditions.

Une fois les conditions correctement paramétrées, on demande de réimporter les messages JMF en erreurs.

 

En détails :

1.  Ajouter une condition sur l’opération (par exemple : Media@Dimension);

2.  Dans la zone « Valeur dans le JMF » saisir une valeur volontairement fausse (par exemple : 123 123) ;

Dans GSTraiteHttp (programme d’importation des messages JMF) les JMF correspondant à cette opération seront affichés en erreur car aucune opération n’est correctement configurée.

3.  Afficher le contenu du message JMF en erreur (en double-cliquant dessus ou clique-droit « Afficher ») et chercher la valeur correspondant à la condition :

a.  Media@Brand : Attribut Brand du nœud Media

b.  Media@Dimension : Attribut Dimension du nœud Media

c.  Media@UserMediaType : Attribut UserMediaType du nœud Media

4.  Saisir dans la valeur trouvée dans la condition ;

5.  Demander de réimporter le message JMF qui était en erreur. Si la condition est correctement définie, le message JMF sera importer et ne sera plus affiché en erreur.
