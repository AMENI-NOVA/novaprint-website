Réalisation d’une sauvegarde avec Graphisoft

Précédent   Suivant

Important : Cette fonction est possible depuis Graphisoft seulement avec les bases de données MSDE  et SQL Server. Pour les autres bases de données, il faut utiliser les outils fournis avec ces dernières.

 

Pour pouvoir effectuer une sauvegarde, l'utilisateur doit avoir le rôle db_backupoperator sur la base de données (ou le rôle db_owner).

 

1. Si vous sauvegardez sur un support amovible (disquette, zip, jaz, cartouche), mettez un support dans le lecteur correspondant.

2. Fermez toutes les fenêtres ouvertes dans Graphisoft : paramétrages, liste des devis, ...

3. Dans la barre de menu, cliquez sur "Fichier", déplacez le curseur sur "Utilitaires" et cliquez sur "Sauvegarde et restauration", pour faire apparaître la fenêtre correspondante.

4. La base de données à sauvegarder sera compressée dans un fichier de sauvegarde, dont vous devez indiquer le nom ainsi que l'emplacement (lecteur de disquettes, lecteur Iomega, disque dur ou autre). Par défaut, Graphisoft vous propose de sauvegarder dans le dossier de travail de l'application, dans un fichier s'appelant "Sauvegarde Graphisoft du (date du jour).svg".

5. Pour modifier l'emplacement de la sauvegarde, cliquez sur le bouton <...>. Une nouvelle fenêtre appelée "Sélectionnez le chemin et le nom de la sauvegarde" apparaît.


Dans la partie supérieure de celle-ci, une liste déroulante libellée "Dans : " permet de sélectionner l'emplacement, parmi les unités disponibles. L'emplacement sélectionné sera automatiquement proposé lors de la prochaine sauvegarde.

6. La liste principale de la fenêtre affiche les sauvegardes déjà présentes sur le support sélectionné.

7. Dans la zone de texte libellée "Nom", vous pouvez saisir le nom de la sauvegarde.

8. Cliquez sur <Enregistrer> pour valider l'emplacement et le nom de la sauvegarde, et revenir à l'écran précédent.

9. Cliquez sur <Sauvegarder>.

10.  A la fin de la sauvegarde, l’onglet "Journal" est affiché dans lequel sont récapitulées les informations relatives à votre sauvegarde : utilisateur, date et heure de début et de fin, chemin du fichier de sauvegarde, erreurs survenues et état de l’opération (réussite ou échec ainsi que la raison de l’échec).
