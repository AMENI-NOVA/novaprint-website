Saisie des temps restants d’une fiche

Précédent   Suivant

La présentation et les informations à saisir diffèrent en fonction du paramétrage du poste :

¨   Sans détail par opération : le temps est saisi globalement pour la fiche

¨   Avec détail par opération : le temps est saisi pour chaque opération

Pour les fiches d’activités secondaires, la saisie est toujours détaillée par opération.

Sans détail par opération

Dans ce cas, le temps restant est global à la fiche, quelles que soient les opérations à effectuer.

¨   Libellé de la fiche : Nom de la fiche de travail. Il est possible de le modifier pour le rendre plus explicite.

¨   Prévu Temps : Correspond au temps restant à effectuer. Si la fiche n'est pas commencée, le temps indiqué est celui prévu par le devis.

¨   Bouton Ok : Sort de l'écran et enregistre les saisies.

¨   Bouton Annuler : Sort de l'écran SANS enregistrer.

Avec détail par opération

Dans le cas d'un poste détaillé, le temps est saisi par opération. Chaque ligne dans le tableau correspond à une opération possible pour ce poste.

Toutes les opérations possibles pour le poste, apparaissent même si elles n’étaient pas prévues initialement. Les opérations pour lesquelles des temps étaient déjà prévus sont placées en tête de liste.

¨   Libellé de la fiche : Nom de la fiche de travail. Il est possible de la modifier pour le rendre plus explicite.

¨   Prévu Nbre : Nombre d'opérations restant à effectuer. Si la fiche n'est pas commencée, le nombre indiqué est celui prévu par le devis.

¨   Prévu Temps : Correspond au temps restant à effectuer. Si la fiche n'est pas commencée, le temps indiqué est celui prévu par le devis.

¨   Bouton Ok : Sort de l'écran et enregistre les modifications éventuelles.

¨   Bouton Annuler : Sort de l'écran SANS enregistrer.
