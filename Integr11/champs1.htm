Champs

Précédent   Suivant

Un champ permet de définir dans le texte un emplacement ou sera inséré une information variable qui dépend de l’édition. Par exemple le numéro de devis est un champ dont la valeur dépend du devis à éditer. Lorsque vous éditez une lettre de confirmation, le champ "Numéro de devis" est remplacé par la valeur correspondante.

Types de champ

Il y a 3 types de champs :

¨   Information non fixe provenant de l'élément à éditer : N° de devis, Nom du client,...
Les informations dépendent du type d'édition.

¨   Libellé lié à un champ d'information et n'étant édité que lorsque l'information liée n'est pas vide. Par exemple, le libellé "Conditionnement : " doit apparaître devant le type de conditionnement, mais ne doit pas apparaître s'il n'y a pas de conditionnement pour le devis édité. Pour chaque champ d'information, il y a un champ libellé lié.
L'utilisation des champs libellés liés n'est pas obligatoire ; on peut taper directement du texte dans le bloc, qui sera alors systématiquement édité.

¨   Marques de répétition : marque un groupe (textes et/ou champ) qui devra être répété par exemple pour des devis liés ou chaque élément d'une facture.
Les marques de répétition sont spécifiques aux types d'édition. Par exemple pour la lettre de confirmation, il y a une marque de répétition pour gérer les devis liés. Pour la facture, il y en a deux : une pour les éléments du document (regroupement de factures), et une pour les taux de TVA à récapituler.
La marque de début de répétition a donc une fonction particulière, ce qui n’est pas le cas de la marque de fin de répétition. Par exemple, dans la liste des champs possibles pour le paramétrage de la facture, il y a deux marques de début et une seule marque de fin.
Attention, dans le modèle, il doit toujours y avoir une marque de fin pour chaque marque de début de répétition.

Insertion d’un champ

Pour insérer un champ dans un bloc, positionnez le curseur à l’endroit voulu, puis cliquez sur Champ dans le menu Insertion ou cliquez sur le bouton Nouveau champ de la barre d’outils. Une boîte de dialogue vous permet alors de sélectionner le champ à insérer parmi ceux proposés. Les champs proposés dépendent du type d’édition. Suivant l’onglet sélectionné, les champs apparaissent triés par ordre alphabétique ou par catégorie.

Mise en forme

Suivant le champ sélectionné, la partie droite de la fenêtre vous permet d’indiquer la mise en forme à appliquer à l’information. Chaque champ accepte une mise en forme qui dépend du type d'information représenté :

¨   Date : plusieurs formats sont proposés. La lettre j  représente le jour, m le mois, a l’année.
Si l’heure est précisée, on pourra l’afficher avec les lettres h et  n.

¨   Texte : minuscules, majuscules, 1ère lettre en majuscules et initiales sont des formats prédéfinis.
Vous pouvez aussi utiliser @ pour placer les caractères, l’affichage du RIB (23 caractères) utilise cette possibilité pour séparer les différents groupes.

¨   Image ou Texte : Image, Image(50%), Image(1,5 ;1,5), Pas de mise en forme,  minuscules, majuscules sont les formats proposés par défaut, mais vous pouvez modifier les valeurs numériques pour adapter l’affichage à votre besoin. L’image insérée est liée au texte et se déplace donc avec lui.

¨   Pas de mise en forme, Minuscules, Majuscules :  Le libellé associé est affiché comme avec la mise en forme de type texte.

¨   Image : l’image associée est affichée dans sa taille réelle.

¨   Image(xxx%) : l’image est affichée avec le zoom indiqué : 50% pour une réduction de moitié. NB : les valeurs admises vont de 10 à 250%. Ex : Image(35%) pou r obtenir une image réduite à 3% de sa taille réelle.

¨   Image(largeur ;hauteur) : l’image est affichée pour entrer totalement dans le rectangle dont les dimensions sont indiquées en cm (en pouces si système US) tout en conservant le rapport de forme. Ex : Image(2,3 ;3,5) pou r obtenir une image qui tiendra totalement dans un rectangle de 2,3 cm x 3,5 cm.

¨   Qualité interlocuteur (M., Mme...) : abrégé ou en entier

¨   Montants et nombres : Plusieurs formats sont proposés, mais vous pouvez utiliser des formats personnalisés.
Mise en forme des champs Prix dans les modèles :

Graphiplus permet de travailler avec plusieurs devises :

Les champs faisant apparaître des prix présentent les mises en formes suivantes :

·    Devise principale : La devise utilisée pour vos tarifs et la comptabilité.

·    Devise secondaire : Votre devise préférée dans les autres devises définies.

·    Devise élément : La devise utilisée pour construire l’élément imprimé. C’est celle paramétrée dans la fiche société du client au moment où son devis a été calculé. La devise élément du devis est transmise à la commande puis à la facture.

·    Autre devise : Si l’élément est en devise principale, il s’agit de la devise préférée parmi les autres devises. Si l’élément est dans une autre devise, c’est la devise principale.

·    "Forme %XXX% où Forme est facultatif et représente un format d’affichage de nombre (avec # , 0 et . ) et XXX représente un symbole international de devise définie dans votre liste de devises: le montant exprimé dans cette devise.
Exemples : "#,##0.0000 %EUR% pour afficher en euros avec 4 décimales.,  %EUR%  pour afficher en euros avec le format habituel, défini dans votre liste de devises.

·    "%XXX% Unité NbDec ss-Unité" : permet d’afficher en lettres le montant avec ‘Unité’ comme texte d’unités, NbDec le nombre de décimales et ss-Unité  le libellé de sous unité.
Exemple : "%EUR% Euros 2 cents" pour afficher en lettre le montant en euros et cents.
Remarque : Ce format n’est disponible que dans certaines langues.

Vous pouvez donc, dans un modèle :

·    Afficher le détail dans la devise de votre client en utilisant la mise en forme « Devise élément » et le total dans une autre devise avec la mise en forme « Autre devise ».

·    Afficher le détail dans votre devise comptable en utilisant la mise en forme « Devise Principale » et le total dans l’autre devise préférée avec la mise en forme « Devise secondaire ».

·    Afficher les prix unitaires avec la précision nécessaire dans une devise choisie.

Par ailleurs, des formats personnalisés sont possibles avec les caractères suivants :

.                  Le premier caractère « . » (point) de la chaîne de format détermine l'emplacement du séparateur décimal dans la valeur mise en forme. Tout caractère « . » supplémentaire est ignoré.

0                 Si la valeur mise en forme possède un chiffre à l'emplacement où «  0 » (zéro) apparaît dans la chaîne de format, ce chiffre est affiché, sinon affiche un zéro à cet emplacement. Si le nombre a moins de chiffres qu’il n’y a de zéros (quel que soit l’emplacement par rapport au séparateur décimal) dans le format, on affiche les 0 devant ou après le nombre.
Exemple : Avec « 0 »  on affiche au moins un nombre  de la valeur arrondie à l’entier le plus proche (0,25 donnera « 0 », « 0123,456 » donnera « 123 »).
Avec « 000 », « 12 » sera affiché « 012 ». Avec « 0.0 » « 12 » sera affiché « 12,0 » (si la virgule est le séparateur décimal du poste).

#                 Si la valeur mise en forme possède un chiffre à l'emplacement où « # » (dièse) apparaît dans la chaîne de format, ce chiffre est affiché, sinon rien n'est affiché.
Exemple : Avec « # »  on affiche aucun ou au moins un nombre  de la valeur arrondie à l’entier le plus proche (0,25 ne donnera aucun affichage, « 0123,456 » donnera « 123 »).
Avec « # », « 12 » sera affiché « 12 ». Avec « #.# » « 12 » sera affiché « 12, » et « 12.35 » sera affiché « 12,4 » (si la virgule est le séparateur décimal du poste).

,                  Le caractère « , » (virgule) remplit deux fonctions. Premièrement, si la chaîne de format contient un caractère « , » entre deux espaces réservés de chiffre (0 ou #) et à gauche de la virgule décimale éventuelle, des séparateurs des milliers seront insérés dans la sortie, entre chaque groupe de trois chiffres à gauche du séparateur décimal.
Deuxièmement, si la chaîne de format contient un ou plusieurs caractères « , » immédiatement à gauche de la virgule décimale, le nombre est divisé par le nombre de caractères « , » multiplié par 1000 avant sa mise en forme. Par exemple, la chaîne de format « 0,, » représente simplement 100 millions par 100. Utilisez le caractère « , » pour indiquer que la mise à l'échelle ne comprend pas les séparateurs des milliers dans le nombre mis en forme. Par conséquent, pour multiplier un nombre par 1 million et insérer des séparateurs des milliers, vous devez utiliser la chaîne de format « #,##0,, ».

%                La présence d'un caractère « % » dans une chaîne de format entraîne la multiplication d'un nombre par 100 avant sa mise en forme. Le symbole approprié est inséré dans le nombre proprement dit, à l'endroit où le caractère « % » apparaît dans la chaîne de format.

 ‘abc’ ou "abc"   Les caractères compris entre des guillemets simples ou doubles sont copiés littéralement dans la chaîne résultante et n'affectent pas la mise en forme.

\                  Ce caractère est utilisé pour forcer l’affichage sans interprétation du caractère qui le suit. Par exemple, pour afficher le symbole # devant un entier on utilisera « \# 0 » qui donnera « # 123 » avec le nombre 123.

;                  Le caractère « ; » est utilisé pour séparer la mise en forme en 3 sections qui seront utilisées pour les nombres positifs, négatifs et les zéros. Vous pouvez utiliser les formats spéciaux définis plus haut et les formats personnalisés pour chaque section. Si une section est omise, c’est la section de la valeur positive qui est utilisée
Exemple : « Devise principale;"négatif ?";# » donnera « 123,50 € » avec « 123.5 » et l’euro, « négatif ? » avec une valeur négative quelle qu’elle soit et n’affichera rien avec une valeur nulle.

¨   Descriptif : les descriptifs de devis, commande, facture etc. sont construits à partir d’un modèle de descriptif paramétrable. Sélectionnez dans la liste des mises en forme le modèle de descriptif que vous souhaitez pour l’édition.

¨   Informations complémentaires : Sélectionnez l’informations complémentaire souhaitée dans la liste des mises en forme ou sélectionnez « Toutes » qui affichera toutes les informations disponibles avec leur libellé et séparées par des retours à la ligne. Voir « Libellé lié à un champ » ci-dessous pour afficher un libellé spécifique d’une information complémentaire sélectionnée.

¨   Libellé lié à un champ : Entrez dans la zone « mise en forme » le texte qui doit apparaître si le champ lié n’est pas vide.

Le texte peut être un code pour ajouter un caractère spécial dépendant de la présence d’informations :

<LF>       Un saut de ligne sera inséré.

<CR>      Une fin de paragraphe sera insérée

<TAB>    Une tabulation sera insérée

<FF>      Un saut de page sera inséré

<CW>     Une césure sera insérée

<NBS>    Un espace insécable sera inséré

Si le champ libellé est lié à un champ qui renvoi un nombre ou un montant alors vous pouvez utiliser le caractères « ; » pour séparer les trois libellés utilisables que vous aurez entourés de « " » :
Exemple :  Avec « Mon libellé », on affichera systématiquement « Mon libellé ».
Avec « "La majoration est " ; "La minoration est " ; "pas de modification" », on affichera  « La majoration est  » si le champ associé renvoi une valeur positive, « La minoration est  » si le champ associé renvoi une valeur négative et « pas de modification  » si la valeur est nulle. Si une section est omise, c’est la section de la valeur positive qui est utilisée.

Si le champ libellé est lié à un champ « « Informations complémentaires » alors indiquez entre &[ et ] le libellé exact de l’information complémentaire à laquelle est associée le libellé en plus du texte que vous voulez afficher.
Exemple :  Avec « Service fabrication :&[ServiceFab]», le texte « Service fabrication : » n’apparaîtra que si l’information complémentaire « ServiceFab » est renseignée.

¨   Codes GS1-128 : Il s’agit de codes utilisés en logistique, composés de plusieurs informations identifiées par un code numérique. Indiquez dans le format du champ ces numéros entre parenthèses pour composer votre code.

o  SSCC (code 00)

o  Numéro de dossier (10)

o  Date du bon de livraison (11)

o  Quantité dans le colis (37)

o  Numéro de commande client (400)

o  Référence article client (241)

Ex : « (00)(10) » pour obtenir le SSCC et le suivi du numéro de dossier

Des exemples non exhaustifs sont présentés dans la liste déroulante.

NB : lorsqu’il n’est pas formaté en code-barres, le code présente les identifiants entre parenthèses alors que lorsqu’il est formaté en code-barres, les parenthèses n’apparaissent pas.

¨   Code-barres : tout champ peut être affiché avec une police codes à barres mais ces affichages nécessitent une mise en forme spécifique au type de codage qui seule permettra à un lecteur de code barres de lire ce que vous avez imprimé. Deux formes de codes barres sont possible :

¨   Le code barre 128 qui permet d’afficher sous forme de code barre tout texte. Utilisez la police « DAR28 », « DB28 » ou « DCR28 » avec cette mise en forme.

¨   Le code barre EAN13 qui permet l’affichage de nombre de 12 chiffres uniquement. Ce codage est utilisé sur tous les articles commerciaux pour les identifier. Utilisez la police « Code EAN13 » pour afficher les champs mis en forme selon ce code.

 

Le même champ peut apparaître plusieurs fois dans le même modèle (ou dans le même bloc) avec une mise en forme différente (ex : "M." dans l'adresse, puis "Monsieur" dans les formules de politesse).


Lorsque vous insérez un champ, celui-ci est représenté par un texte entre deux accolades, qui sera remplacé par l’information correspondante au moment de l’édition du document. Si vous déplacez le curseur de la souris sur un champ, le curseur est remplacé par une flèche et si vous cliquez, le texte représentant le champ apparaît alors en gris foncé.
Pour avoir une meilleure représentation de votre paramétrage, vous pouvez remplacer ce texte par défaut par celui de votre choix. Veillez toutefois à ne pas effacer complètement le texte qui représente le champ, car sinon c’est le champ lui-même qui sera supprimé du bloc. Si vous changez le texte représentant un champ, le nouveau texte doit également apparaître en gris foncé indiquant ainsi qu’il symbolise le champ et qu’il sera bien remplacé par l’information correspondante.
