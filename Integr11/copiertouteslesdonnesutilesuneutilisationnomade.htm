Copier toutes les données utiles à une utilisation nomade

Précédent   Suivant

Le but de ce traitement est d'initialiser les données nomades de l'ordinateur sur lequel le programme est lancé.

Important : Cette fonction nécessite que Graphisoft soit démarré à partir de la base de données du réseau.

Si vous avez travaillé sur les données "nomades", vous devez d'abord synchroniser le serveur avec les données de votre poste. Voir "Transfert des nouvelles données entre la base nomade et la base réseau ".

 

Une fois connecté à la base nomade, il faut définir (ou vérifier) le préfixe des devis pour la base nomade.

Ensuite sélectionner les éléments à transférer.

Puis, il faut cliquer sur "Copier" pour lancer le traitement.

 

Préfixe des devis "nomades"

Par défaut, le programme vous propose le préfixe saisi lors de la précédente copie.

Le préfixe devant un numéro de devis sert à identifier la base de données sur laquelle à été fait le devis. Par exemple, si votre base nomade utilise le préfixe "A", tous les devis enregistrés commenceront par A.

Il est fortement recommandé d'indiquer un préfixe différent de celui de la base de données du réseau. Cela permet de ne jamais avoir le même numéro de devis entre le réseau et la base nomade, ce qui évite des conflits de numérotation lors de la récupération des devis. Un conflit oblige à re-numéroter le devis.

Etendue du transfert

Toutes les données utiles pour une utilisation nomade de Graphisoft sont transférées :

¨   Le paramétrage général de Graphisoft;

¨   Le paramétrage du calcul du devis;

¨   Le paramétrage de la messagerie;

¨   La liste des sociétés;

¨   La liste des devis (plus les lettres de confirmations);

¨   La liste des commandes (plus les accusés réception et les pochettes de fabrication);

¨   La liste des notes des sociétés et des devis;

¨   La messagerie;

¨   La gestion d'informations.

 

Les données suivantes NE SONT PAS TRANSFERES dans la base nomade :

¨   Les factures;

¨   Les données de gestion de production (bons de commande ou de livraison, avancement des commandes, temps passés, …);

¨   Les données de gestion de stocks.

Cela signifie que les listes et les fonctions utilisant ces données seront vides ou incomplètes.

 

La durée de la copie est variable en fonction du volume des données et de la vitesse de l'ordinateur. Pour accélérer ce traitement il est possible de restreindre le volume des données à copier. Pour cela, il faut utiliser les options situées dans "Etendue du transfert".

 

Sociétés

Toute la liste des sociétés est transférée dans la base nomade. Il n'est pas possible de la restreindre.

Devis

Tous les devis n'étant pas utiles dans la base nomade et pour accélérer le traitement il est possible de restreindre la liste des devis transférés. Pour cela, deux critères cumulables sont disponibles :

¨   Deviseur / Commercial : soit tous les devis, soit les devis où l'utilisateur est le deviseur ou le commercial

¨   Date : soit tous les devis, soit les devis créés durant les derniers mois (la liste déroulante permet de choisir le nombre de mois).

Notes

Le nombre de notes et le volume de leurs pièces jointes pouvant vite devenir important, il est très utile de restreindre la liste des notes à transférer. Pour accélérer le transfert des notes, voici les quatre critères cumulables disponibles :

¨   Sociétés : Soit les notes de toutes les sociétés, soit seulement les notes des sociétés où l'utilisateur est le commercial.

¨   Devis : Soit les notes de tous les devis transférés, soit seulement les notes des devis où l'utilisateur est le deviseur ou le commercial

¨   Pièce jointes : Soit toutes les pièces jointes des notes transférées, soit seulement les pièces jointes où la taille est inférieure à celle indiquée dans la liste déroulante. Il est impossible de transférer des pièces jointe supérieure à 2 Mo.

¨   Date : Soit toutes les notes, soit seulement les notes créées ces derniers mois (la liste déroulante permet de choisir le nombre de mois)

 

Copie des fichiers

A tout moment, il est possible d'arrêter la copie en cliquant sur le bouton "Arrêter". Mais si vous répondez "Oui", la base de données nomade n'est plus utilisable : il faut relancer la synchronisation ou restaurer une sauvegarde.

 

Une fois la copie terminée, vous pouvez vous déconnecter du serveur et travailler avec les données copiées sur le poste.
