Description du fichier d’import

Précédent   Suivant

Le fichier d’import présente les informations en colonnes avec une ligne d’entête qui contient les noms des colonnes. Ce sont ces noms qui permettent à la fonction d’import de reconnaître les colonnes de votre fichier.

NB : Les colonnes marquées d’un * sont obligatoires.

Description de l’adresse à livrer :

·      "Nom client"* pour la colonne du nom du destinataire

·      "Adresse"* pour la colonne d’adresse

·      "Code postal" pour la colonne Code postal. Attention : si la valeur est numérique elle sera formatée sur 5 chiffres lors de la lecture pour éviter la perte éventuelle du 0 devant la valeur.

·      "Ville"* pour la colonne Ville

·      "Etat" pour la colonne Etat

·      "Pays" pour la colonne Pays. Attention : le libellé du pays doit figurer dans le paramétrage Graphisoft des pays.

Les adresses seront éventuellement créées « Non favorite » dans les adresses de livraison du client concerné par le bon de livraison.

Description de la livraison à réaliser :

·      "Famille Livraison" et "Point Livraison" pour les colonnes définissant un point de livraison. Attention : les libellés doivent figurer dans le paramétrage Graphisoft des familles et points de livraison. Si le point n’existe que dans une seule famille, il est possible d’omettre la famille.

·      "Transporteur" pour la colonne Transporteur. Attention : le libellé du transporteur doit être le nom interne d’une société sous-traitante de transport paramétrée dans Graphisoft.

·      "Consignes" pour la colonne Consignes du bon de livraison

·      "Nbre colis" et "Colis" pour les deux colonnes décrivant un sur-conditionnement si plusieurs références sont livrées par le bon de livraison. Attention : le libellé du conditionnement doit figurer dans le paramétrage Graphisoft des conditionnements.

Description des objets à livrer :

·      "Quantité"* pour la colonne Quantité. Ajouter l’indice 1,2,3… si plusieurs références

·      "Référence"* pour la colonne Référence. Ajouter l’indice 1,2,3… si plusieurs références

·      "Nbre colis" et "Colis" pour les deux colonnes décrivant un colisage. Ajouter l’indice 1,2,3… si plusieurs références. Attention : le libellé du conditionnement doit figurer dans le paramétrage Graphisoft des conditionnements.
NB : Si ces colonnes sont répétées, elles décrivent plusieurs conditionnements imbriqués pour la référence concernée (Ex : sous film puis en carton). Dans ce cas, l’ordre du plus petit au plus grand doit être respecté. (Ex : placer les colonnes « sous film » à gauche des colonnes « En carton »)

Exemples de fichier :

-   Adresse du destinataire, consigne et une référence à livrer :

Nom client

Adresse

Code postal

Ville

Consignes

Quantité

Référence

M. Martin

12, rue du val

45000

Orléans

Le matin

1500

2013120123

Dans ce cas, c’est la livraison « à répartir » qui recevra tous les bons de livraison. Définissez pour cette livraison l’expéditeur, le transporteur et, avec l’option « colisage », les conditionnements (types, quantité par paquet et  modèles d’étiquettes).

 

-   Pour décrire plusieurs références, indicez les colonnes quantité et référence :

Quantité 1

Référence 1

Quantité 2

Référence 2

1500

2013120012

3000

2013120156

-   Pour décrire des conditionnements ajoutez  les colonnes « nbre colis » et « colis » indicées comme chaque référence :

Nbre colis 1

Colis 1

Nbre colis 1

Colis 1

Quantité 1

Référence 1

50

En carton

1

Sur palette

5000

2013110025

Ainsi, les conditionnements spécifiés dans les premières colonnes « Nbre colis 1 » et « Colis 1 » seront placés dans les conditionnements spécifiés dans les colonnes suivantes. Si un nombre de colis est laissé à vide ou à 0, il sera ignoré.

Un dialogue vous permettra de spécifier le modèle d’étiquette à utiliser pour chaque type de conditionnement rencontré dans le fichier.

 

-   Pour ajouter un sur-conditionnement de l’ensemble des références d’un bon de livraison, ajoutez les colonnes « nbre colis » et « colis » sans les indicer.
NB : si la colonne « Nbre colis » est présente sans la colonne « colis » pour le sur-conditionnement, un dialogue vous permettra de sélectionner le type de sur-conditionnement et le modèle d’étiquette à utiliser.
Le sur-conditionnement n’est pris en compte que si plusieurs références sont groupées dans un même bon de livraison.

 

Pour renseigner les références à livrer, utilisez le numéro d’une commande Graphisoft. Si la commande a plusieurs versions, utilisez le numéro de la commande suivi du nom de la version ou de sa référence article client séparés par « - » (espace tiret espace). Ex : « 2013120056 – Version anglaise »

Les différentes références d’une ligne du fichier peuvent être différentes versions d’une même commande ou de commandes différentes, mais le client concerné sera défini comme le client de la première référence apparaissant sur le bon de livraison.
