Organisation

Pour être rapidement accessibles les articles sont organisés en types d’articles, puis en familles.

Les types d’articles sont des groupes de familles (papiers, encres, plaques…).

Les familles sont des groupes d'articles possédant les mêmes caractéristiques générales.

Exemple : Tous les grammages et formats d'un papier possèdent les mêmes caractéristiques générales; donc la famille sera le papier et les articles de cette famille seront les grammages et les formats de ce papier.

Les lots sont des subdivisions des stocks. Ils permettent de définir l’origine précise des matières, leurs positions dans l’entrepot, éventuellement leur conditionnement. Ils sont repérés par un code jusqu’à 30 caractères qui peut être imprimé sur des étiquettes sous forme de code à barres.

Codage

Les familles sont représentées par un code (raccourci) possédant 12 caractères au plus, les articles par un code de 12 caractères au plus.

Le code complet de l'article est l'ensemble du code de sa famille et de son code propre mis bout à bout.

Exemple : Pour les papiers, la famille est représentée par un code formé des codes catégorie et papier utilisés dans le fichier des papiers (voyez le  fichier papier et le paragraphe "Création des articles - Articles papier" plus loin). Donc  pour le Magnoprint, si le code catégorie est ‘CMB7 et le code papier ‘MAPM’, les grammages et formats du magnoprint seront tous dans la même famille de code ‘CMB7MAPM’ et de nom Magnoprint. Le code article est composé du grammage et du format ce qui pourra donner: ‘  90  45  64’.
Le code complet de l'article sera donc: ‘CMB7MAPM      90  45  64’. Pour un article plaque, on peut avoir: ‘PLAQNEGA      70  67  94’ où 70 représente l'épaisseur de la plaque en microns et 67x94 son format.

Ainsi, la structure du code de l'article permet de préciser de gauche à droite de quel article il s'agit jusqu'à ce qu'il n'y ait plus d’ambiguïté possible sur la nature exacte de l’objet représenté.

La saisie des 24 caractères du code complet d’un article étant source d'erreurs, la spécification d'un article est divisée en une zone ‘code famille’ puis une autre ‘code article’ dans cette famille.

Possesseurs

Un stock est défini comme un article muni d’une précision supplémentaire indiquant son appartenance à un possesseur. Les possesseurs ayant un code de quatre caractères alphanumériques, le ‘code stock’ fait 28 caractères au total.

A chaque fois que l'utilisateur devra spécifier un article ou un stock, pour indiquer une entrée ou une sortie par exemple, il devra entrer le code complet de l'article et préciser le possesseur. C'est à dire taper le code de la famille, puis le code de l'article et le code du possesseur.

Si les stocks de l’article sont gérés par lots, l’entrée d’un code lot à la place d’un code article définira à la fois le code article, le code possesseur et la référence du lot.

Recherche d'un stock

Il est possible de faire afficher une fenêtre d'aide qui permet de choisir parmi la liste des articles ou des familles pour en sélectionner un.

Ce découpage peut sembler long mais chercher une famille parmi 100 puis un article parmi 100 puis un stock parmi 5 sera plus rapide que de chercher un stock dans une liste de 50,000 stocks.

Si on doit sélectionner un lot, cela se fera dans une fenêtre particulière, mais généralement on utilisera un lecteur de code à barres pour saisir le code d’un lot. Ceci remplace alors la saisie du code du stock.

La liste des stocks permet aussi de rechercher un stock mais les critères disponibles ne sont pas les mêmes.

Liste des stocks

Cette liste permet de créer/modifier/supprimer les familles et les articles de stocks. Comme les autres listes de l’application, elle permet de définir des macros et/ou d’utiliser des filtres express.
