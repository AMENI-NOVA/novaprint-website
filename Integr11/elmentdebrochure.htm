Types d'élément de brochure

De même que les devis sont regroupés en types, les éléments d'une brochure peuvent être de différents types selon le type standard paramétré pour le brochage sélectionné pour le devis.

Cahier

Pour tous les éléments de ce type, on considère que le pliage est fait en plis croisés. Vous avez la possibilité pour chaque cahier, de modifier le format fini (une question vous est posée après la saisie du nombre de pages).

Couverture

Ce type d'élément est traité de la même manière qu'un encart. Il faut cependant différencier la couverture car dans certains cas le pliage de celle-ci ne doit pas être compté :

¨   Brochures 2 piqûres à cheval, si l'encarteuse piqueuse peut plier la couverture (Voir « Encartage piquage » dans le chapitre « Paramétrages de calcul des devis »).

¨   Brochures en Dos carré collé ou Couture fil, le pliage étant fait en même temps que l'encollage à chaud.

Ce type d’élément est indisponible lorsque le brochage sélectionné est de type standard « Couverture rigide ». Dans ce cas, la couverture rigide se compose de plusieurs éléments :  Plat, Dos et Matière de couverture (voir ci-dessous).

Encart broché

Les encarts brochés (comme d'ailleurs les couvertures) sont des éléments qui sont traités comme des dépliants (Voir chapitre Dépliant). Ils sont pris en compte dans le calcul du brochage

Encart jeté

Les encarts jetés sont calculés comme les encarts brochés, mais ne comptent pas dans le calcul du brochage.
Contrairement aux couvertures et aux encarts brochés, le format d’imposition d’un encart jeté ne comprend ni rogne, ni dépassant, quel que soit le paramétrage du chemin de brochage utilisé. Mais vous avez la possibilité d’indiquer s’il y a ou non un fond perdu en utilisant la liste de choix correspodante (voir « Elément de dépliant et d’affiche/fiche » plus lon dans ce chapitre).
Si l’encart jeté fait au moins 4 pages, le pliage est toujours calculé, même si le chemin de brochage utilisé ne comporte aucune plieuse.

Types d’éléments spécifiques aux livres à couverture rigide

La couverture rigide se compose de plusieurs types d’éléments :

¨   Plat de couverture : obligatoire

¨   Dos de couverture : optionnel

¨   Matière de couverture : obligatoire

Ces types d’éléments ne sont accessibles que lorsque le type de façonnage sélectionné a comme type de brochage standard « Couerture rigide (HardCover) ».

Plat de couverture

Pour ce type d’élément, il ne peut pas y avoir d’impression, d’opération typo ou d’opération de surface : les saisies correspondantes sont inaccessibles.

¨   Calcul automatique des dimensions

Les dimensions sont calculées automatiquement en fonction du format de l’intérieur du livre et des paramètres du type de façonnage spécifiques aux couvertures rigides :

·    Largeur = Largeur intérieur + Ecart largeur plat/intérieur

·    Hauteur = Hauteur intérieur + Chasse

Le format à plat calculé est modifiable par l’utilisateur.

Les données d’optimisation de ce type d’élément permettent de modifier la valeur provenant du type de façonnage, pour la chasse et l’écart entre la largeur du plat et de l’intérieur.

¨   Modèles avant, arrière et dos de la couverture

Vous pouvez indiquer si l’élément correspond au(x) plat(s) de couverture avant et/ou arrière.

Il est aussi possible d’amalgamer dans le même élément, le dos de la couverture, mais dans ce cas, le module GsGang est nécessaire pour effectuer le calcul de l’amalgame. L’amalgame du dos avec le plat n’est possible que s’il n’y a pas d’élément de type « Dos de couverture ».

NB : GsGang n’est pas nécessaire pour calculer l’amalgame des plats de couverture avant et arrière.

Si le livre comporte des plats de couverture différents pour l’avant et l’arrière, des éléments distincts doivent être saisis.

Dos de la couverture

Pour ce type d’élément, il ne peut pas y avoir d’impression, d’opération typo ou d’opération de surface : les saisies correspondantes sont inaccessibles.

Le dos de la couverture peut être amalgamé avec le plat de la couverture (voir « Plat de couverture »).

¨   Calcul automatique des dimensions

Les dimensions sont calculées automatiquement en fonction du format fini du livre et des paramètres du type de façonnage spécifiques aux couvertures rigides :

·    Largeur = Epaisseur feuilles intérieur x Nombre de pages intérieur / 2

·    Hauteur = Hauteur format intérieur + Chasse

Le format à plat calculé est modifiable par l’utilisateur.

Matière de couverture

Pour ce type d’élément, on peut saisir de l’impression, des opérations de typo ou de surface.

¨   Calcul automatique des dimensions

Les dimensions sont calculées automatiquement en fonction du type de dos et des dimensions du plat et du dos de la couverture :

·    Type de dos plat :

Largeur = (Rembordage côté x 2) + (Epaisseur Plat x 2) + (Espacement plat/dos x 2)
               + (Largeur plat x 2) + Largeur Dos + (Epaisseur Dos x 2)

·    Type de dos arrondi :

Largeur = (Rembordage côté x 2) + (Epaisseur Plat x 2) + (Espacement plat/dos x 2)
               + (Largeur plat x 2) + Largeur Dos

·    Hauteur = Hauteur Plat + (Rembordage haut x 2) + (Epaisseur Plat x 2)

Les données d’optimisation de ce type d’élément permettent de modifier la valeur provenant du type de façonnage, pour le rembordage côté, le rembordage haut et l’espacement plat/dos.

Pages de garde

Ce type d’élément est disponible pour les livres à couverture rigide, mais aussi pour les brochures ayant un type de façonnage dont le type de brochage est « Dos carré collé » (soft cover).

Les pages de garde sont optionnelles dans tous les cas.

Comme pour l’élément « plat de couverture » on peut indiquer que l’élément « pages de garde » comprend la page de garde avant et/ou arrière. On considère dans ce cas qu’elles sont identiques. Si les pages de garde sont différentes (papier, impression, …), l’utilisateur devra créer un type d’élément distinct pour chaque page de garde.

Par défaut, la dimension des pages de garde est intialisée en fonction du format fini de l’intérieur :

¨   Largeur = Largeur format fini x 2

¨   Hauteur = Hauteur format fini

Nombre de pages

Pour les éléments de type 'Cahier', le nombre de pages doit être un multiple du plus petit cahier possible en fonction du type de brochage sélectionné.

Le nombre de pages du plus petit cahier possible pour un type de brochage est déterminé d'après les liaisons existantes entre les chemins de brochage se trouvant dans le type de brochage, et les différents types de cahiers.

Pour les chemins de brochage n'étant liés à aucun type de cahier, on considère par défaut que le plus petit cahier possible est le 4 pages.

Par exemple :

¨   Si tous les chemins de brochage du type de brochage sélectionné ne sont liés à aucun type de cahier, le nombre de pages d'un élément de type 'Cahier' devra être un multiple de 4.

C'est le cas le plus courant, avec les brochures piquées à cheval.

¨   Si au moins un des chemins de brochage du type de brochage sélectionné est lié au type de cahier de 2 pages, le nombre de pages d'un élément de type 'Cahier' pourra être un multiple de 2.

C'est le cas pour les brochures en dos carré collé par exemple.

 

Graphisoft vous avertit si le nombre de pages saisies n'est pas compatible avec le type de brochage que vous avez sélectionné. Vous devez alors, soit :

¨   modifier le nombre de pages

¨   sélectionner un autre type de brochage permettant de réaliser le cahier comportant ce nombre de pages

¨   modifier dans votre paramétrage, les liaisons entre les chemins de brochage du type de brochage sélectionné et un type de cahier compatible avec le nombre de pages saisi pour l'élément (reportez-vous au paragraphe "Paramétrage des types de cahier" du chapitre "Paramétrages de calcul des devis").

Pour calculer l'impression d'un cahier, le nombre de pages sera divisé suivant des types de cahiers connus par le logiciel. Ainsi, un cahier de 44 pages pourra donné à l'impression : 1 cahier de 32 pages + 1 cahier de 8 pages + 1 cahier de 4 pages (ou 2 x 16 pages + 1 x 8 pages + 1 x 4 pages). Vous pouvez cependant imposer une solution pour les types de cahier à utiliser, dans la partie "Optimisation".

Format à plat (couverture et encart)
Calcul automatique du format à plat

Un format à plat calculé automatiquement vous est proposé.

Le format calculé tiendra compte de l'épaisseur du dos, calculé à partir des éléments précédemment saisis, si le paramètre "Calcul automatique de l'épaisseur du dos de la couverture" a été coché pour le type de brochage sélectionné (voir "Paramétrage des types de façonnage" dans le chapitre "Paramétrages de calcul des devis"). Il est donc préférable de saisir d'abord les éléments de type "Cahier", pour finir par ceux de type "Encart" ou "Couverture".
Le calcul automatique de l’épaisseur du dos est effectué en additionnant l’épaisseur de chaque élément du devis, calculée en fonction du nombre de pages et de l’épaisseur du papier sélectionné pour l’élément. Si l’épaisseur du papier n’est pas renseignée, elle est calculée à partir du grammage sur la base d’une main par défaut de 1,7. (1,7 cm pour 100 feuilles de 100g/m2 : par exemple pour une brochure comprenant un intérieur de 500 pages en 80 g/m2, l’épaisseur sera de : 500 / 2 x 1,7 / 100 x 80 / 100, soit 3,4 cm). Si une catégorie de papier a été sélectionnée pour l’élément et qu’elle contient des papiers avec des épaisseurs différentes, l’épaisseur retenue pour le calcul est l’épaisseur moyenne de tous les papiers de cette catégorie.

Si vous modifiez le nombre de pages de l’intérieur, le format à plat de la couverture sera automatiquement recalculé en fonction de la nouvelle épaisseur de la brochure. Si vous travaillez sur un devis qui a été rappelé, Graphisoft vous demandera d’abord de confirmer le recalcul automatique du format, au cas où celui-ci aurait été modifié initialement pour tenir compte par exemple d’un rabat .

Lors du calcul ce format sera augmenté des rognes (Voir "Paramétrage des chemins de façonnage" dans le chapitre "Paramétrages de calcul des devis") sur 4 côtés, et du dépassant suivant le chemin de brochage utilisé pour le calcul (ceci ne concerne pas les encarts jetés).

Le format à plat est automatiquement recalculé si vous modifiez le format fini, l'orientation, le type de brochage ou le nombre de pages de l'intérieur.

Sens des valeurs par rapport au côté du brochage

Le sens des valeurs par rapport au côté du brochage détermine l’affichage du pli sur le schéma d’imposition. Il est déterminé de la manière suivante :

1. Si une des 2 valeurs du format à plat est égale au format fini de la brochure, c’est cette indication qui détermine en priorité le côté qui du brochage.

Ex : pour une brochure 21 x 29,7 (à la française), si la couverture a un format à plat de 29,7 x 42, c’est le côté 29,7 qui est celui du brochage. Idem si le format à plat de la couverture est 42 x 29,7

2. Si aucune des 2 valeurs du format à plat n’est égale au format fini de la brochure, on considère que les valeurs du format à plat ont été saisies dans le même sens que le format fini. C’est donc toujours la 2ème valeur qui correspond au côté du brochage.

Rabats de couverture

Les rabats de couverture peuvent être saisis pour les brochures dont le type de façonnage est lié à « Piqûre à cheval » ou « Dos carré collé » (par l’intermédiaire du type de brochage).

Lorsque la valeur d’un rabat est modifiée, Graphisoft propose de modifier le format à plat de la couverture en correspondance avec la valeur des rabats.

Le format à plat de la couverture inclut les rabats. Le calcul du devis est basé sur le format à plat de la couverture.

Si le nombre de pages (ou le papier) est modifié, Graphisoft propose de recalculer le format à plat de la couverture et tient compte de la valeur des rabats.

Calcul du pliage (couverture et encart)

Cette case à cocher vous permet d'indiquer au moteur de calcul si le pliage doit être calculé pour cet élément.

Par exemple, si vous devisez une brochure comportant un encart qui vous est livré imprimé et plié, décochez cette case pour que le pliage de cet élément ne soit pas calculé.

Dans le cas d'une brochure piquée à cheval dont le brochage est fait sur une encarteuse pouvant plier les couvertures, si vous décocher cette case pour la couverture, aucun pliage ne sera compté et la couverture sera considérée comme un cahier comme les autres pour l'encartage, c'est-à-dire que la majoration "Pliage de couverture" de l'encarteuse ne sera pas comptée non plus.

Attention, si des rabats existent pour une couverture, celle-ci ne peut être pliée par l’encarteuse.

Référence

Il s’agit d’un texte libre vous permettant de donner une description plus détaillée de chaque élément.

Sens des fibres

Cette option vous permet de préciser si l’orientation des fibres du papier a une importance par rapport au format fini ou à plat.

Si c’est le cas, vous devez indiquer si les fibres du papier doivent être parallèles au grand ou au petit côté du format fini ou à plat.

Le format pris en compte (fini ou à plat) dépend du type d’élément :

¨   format fini pour les cahiers

¨   format à plat pour les couvertures et encarts.

Le sens des fibres est pris en compte pour les feuilles (impression offset, typo ou sérigraphie) et pour les bobines (impression offset, numérique ou sérigraphie avec alimentation en bobine).

La détermination du sens des fibres du papier s’effectue selon les règles suivantes :

¨   Pour les papiers en feuilles, on considère que les fibres sont parallèles à la 2ème valeur saisie pour le format de la feuille livrée par le papetier. Il s’agit généralement du côté le plus grand, mais ce n’est pas toujours le cas et si vous créez un format de papier en saisissant 92/65, le programme considèrera que les fibres sont parallèles au côté « 65 ».

¨   Pour les papiers en bobine, les fibres sont parallèles au sens de défilement de la bobine.
