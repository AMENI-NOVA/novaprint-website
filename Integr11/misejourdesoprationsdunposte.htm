Mise à jour des opérations d'un poste

Précédent   Suivant

La liste des opérations est affichée en double-cliquant sur le poste sélectionné. Les opérations sont listées par ordre alphabétique.

Les actions suivantes sont possibles :

¨   Insérer une nouvelle opération pour le poste (voir les caractéristiques d'une opération ci-dessous).

¨   Modifier une opération (voir les caractéristiques d'une opération ci-dessous).

¨   Supprimer une opération. Si une opération existe avec le même libellé pour plusieurs postes, sa suppression pour le poste en cours n’entraîne pas la suppression pour tous les postes où elle apparaît.

¨   Dupliquer: Cette fonction permet de dupliquer une ou l'ensemble des opérations du poste (poste ‘source’) vers un ou tous les postes de ce service (postes ‘destination’).
Dans le cas où toutes les opérations sont à dupliquer, deux choix sont possibles :

·    Compléter : les opérations du poste ‘source’ n'existant pas dans les postes ‘destination’ seront ajoutées à ceux-ci, sans modification des opérations déjà existantes.

·    Remplacer : Ce traitement n’est possible que si pour le poste ‘destination’, aucune fiche de travail n’utilise une de ses opérations. Aussi, l’application écartera automatiquement du traitement les postes ‘destination’ dans ce cas et un message à la fin du traitement vous en donnera la liste.

Pour chaque poste ‘destination’ pour lequel le traitement peut être effectué :

·    Toutes les opérations du poste ‘destination’ seront supprimées.

·    Toutes les opérations du poste ‘source’ seront ‘recopiées’ dans le poste ‘destination’.

·    A la fin du traitement, le poste ‘source’ et les postes ‘destination’ non écartés du traitement auront exactement les mêmes opérations, et avec les mêmes numéros d'ordre.

·    Pour les postes écartés de ce type de duplication, il vous est possible d’effectuer une duplication mais en utilisant le choix ‘compléter’.

Caractéristiques d'une opération

 

 

Nom de l'opération

Une opération peut apparaître pour plusieurs postes, avec le même nom. Mais ce nom doit être unique parmi les opérations d’un poste.

En cas de modification du nom d'une opération pour un poste :

¨   Les autres postes conservent l'opération avec le nom d'origine.

¨   Si des contrôles sont liés à cette opération (avant modification) et au service du poste par lequel on a sélectionné l'opération, le programme demande si les contrôles doivent désormais être liés au nouveau nom de l'opération.

Numéro d'ordre

Ce numéro permet de trier les opérations au sein d'un poste, pour les faire apparaître dans un ordre logique. Par exemple pour le traitement d'une fiche de travail, il est souhaitable que les opérations soient affichées dans l'ordre dans lequel elles sont généralement exécutées.

Le numéro d'ordre est aussi utilisé par la gestion des points de contrôle, lorsqu'un contrôle bloquant est lié après une opération. Toutes les opérations dont le numéro d'ordre est supérieur ne pourront être démarrées que lorsque le contrôle aura été validé (voir "Contrôles bloquants" dans le chapitre "Contrôle de production et planification".

Opération affectable (à un dossier)

¨   Opérations affectables : ces opérations concernent obligatoirement un dossier client.

¨   Opérations non affectables : il s'agit des activités dites secondaires, telles que la réparation, l'entretien, ou toute autre opération ne dépendant pas d'un dossier.
Lorsqu'un poste est créé par la fonction d'initialisation automatique par le devis, une opération non affectable "Activité secondaire" est créée par défaut.

Type d’opération

Trois types d’opération sont possibles

·      Calage ;

·      Production ;

·      Lavage

Si le type d’opération est à « Non défini », celui-ci est derterminé automatiquement en fonction des opérations « devis » qui alimentent cette opération.

 

Ce paramétrage est utilisé, entre autre,  par Emaplan et l’export JDF.
