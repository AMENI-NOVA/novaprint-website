Code

Ce code à usage interne au logiciel, permet d’identifier de manière unique chaque utilisateur. Par défaut, le nom de l’utilisateur est proposé. Ce code est indispensable au cas où deux employés aient le même nom et le même prénom. Lors de la création, c'est le nom de l'employé qui vous est proposé par défaut. Le code peut contenir des caractères alphanumériques.

Type

Le type permet de classer les employés suivant la manière dont ils utilisent ou non l’application.

¨   Utilisateur classique : employés utilisant l’application, exceptés ceux n’utilisant que les fonctions des terminaux de l’atelier.

¨   Administrateur : utilisateur gérant les paramétrages et particulièrement les droits d’accès aux fonctionnalités de l’application. L’administrateur ne correspond pas à une personne physique de l’entreprise, mais permet d’isoler l’accès aux paramétrages. L’administrateur est créé automatiquement dès l’instant où le fichier des employés n’est pas vide. On ne peut créer qu’un seul administrateur et il ne peut pas être supprimé.

Un administrateur a accès aux fonctionnalités suivantes :

·    Paramétrage : Paramétrages généraux et Utilitaires

·    Devis : accès à la mise à jour des fichiers

·    Graphifab, Graphifab+, Intégrale : Paramétrages et Archivage

·    Graphistock : Création, modification et suppression des stocks, Mise à jour des noms et des abonnements, Archivage, Publication y compris la sortie des réservations, Paramétrage.

¨   Atelier : employés utilisant les fonctionnalités des terminaux atelier de la gestion de production.

¨   Autre : employés de l’entreprise n’utilisant pas l’application mais devant être référencés (certains commerciaux par exemple).

¨   GS CRM : utilisateurs n'ayant accès qu'aux fonctions de GS CRM.

Ce type d'utilisateur est lié à un type de licence particulier permettant d'utiliser GS CRM sans avoir à acquérir un droit administratif. Vous ne pouvez enregistrer des utilisateurs de ce type qu'en fonction du nombre de droits GS CRM que vous possédez.

Ces utilisateurs ne pourront pas avoir accès aux fonctionnalités de Graphisoft hors la partie GS CRM, quels que soient les droits d'accès paramétrés.Lorsqu’une identification de l’utilisateur est nécessaire au démarrage de l’application, seuls les employés de type "Utilisateur  classique" et "Administrateur" apparaissent dans la fenêtre d’identification.

Mot de passe

Il est possible d'attribuer à chaque utilisateur un mot de passe qui sera demandé à chaque démarrage, lors de l'identification.
Il permet d'empêcher un utilisateur de s'identifier sous un autre nom, pour bénéficier de droits dans l'application qu'il n'a pas.

Il n'est pas indispensable pour les utilisateurs n'ayant que peu de droits, par contre il est indispensable pour les utilisateurs ayant accès à des fonctions que vous voulez protéger (voir "Droits d’accès").

Appartient à l’atelier

Cette option sert à indiquer les employés qui utilisent l’application à la fois depuis un poste de travail de bureau, et depuis les terminaux de l’atelier. Cette option est toujours cochée pour les employés de type "Atelier", car elle est redondante dans ce cas.

Terminal Atelier : Saisie du mot de passe systématique

(Seulement si l’employés appartient à l’atelier)

Cette option améliore la sécurité et supprime des risques d’erreurs lors de la sélection des employés dans le terminal Atelier.

Si cette option n’est pas cochée, le mot de passe est demandé seulement si l’employé vient d’arriver.

Si cette option est cochée, le mot de passe est demandé à chaque fois que l’employé choisi son nom dans le terminal atelier.

De plus, le fait de sélectionner cette option force le terminal atelier à revenir automatiquement à la liste des employés après une période d’inactivité (paramétrable dans les options du terminal).

Est commercial ou vendeur

Cette option sert à indiquer les employés qui doivent figurer dans la liste des commerciaux qui apparaît dans différentes fonctionnalités de l’application.

Coefficient minimum autorisé

Cette zone permet de définir un coefficient minimum autorisé pour les devis.

Ce coefficient minimum est utilisé dans :

¨   La saisie de devis : Impossible de calculer (donc d'enregistrer) un devis si le coefficient saisi est inférieur au coefficient minimum de l'utilisateur

¨   Les propriétés de devis et le devis simplifié : Impossible de saisir un coefficient inférieur au coefficient minimum de l'utilisateur.

Fonction

Voir le chapitre précédent. La sélection d’une fonction n’est pas obligatoire. Elle ne sert que pour les employés utilisant les terminaux de l’atelier.

Poste par défaut

Indiquez selon sa fonction,  le poste de travail habituel de l'employé, qui lui sera proposé par défaut à chaque identification, au début de la journée.

Prix de l'heure payé

Ce prix d'heure pourra être utilisé pour calculer le coût réel, en fonction de l'occupation des postes par les employés pour chaque dossier.

E-mail (adresse électronique)

Le nom et l’adresse électronique de l’utilisateur courant peuvent être ajoutés au corps du message lorsque vous envoyez un document par courrier électronique dans Graphisoft avant les coordonnées complètes de l’imprimerie. L’adresse électronique particulière du vendeur et de l’utilisateur peuvent être ajoutées aux documents édités dans Graphisoft.

Gestion des stocks

NB : Ce cadre n’est actif que si vous avez le module GS Ware.

Indiquez le code du tampon de saisie des écritures des stock utilisé par défaut par cet employé et cochez la case si les commandes de stocks générées par la création de commande client doivent y être placées par défaut. Dans le cas contraire, elles seront placé dans le tampon indiqué par les paramétrages généraux de la Gestion des stocks.
