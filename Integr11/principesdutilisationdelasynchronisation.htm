Principes d'utilisation de la synchronisation

Précédent   Suivant

Si le poste nomade peut être connecté au serveur :

¨   Avant de se déconnecter du serveur de base de données, l'utilisateur demande à mettre à jour les données utiles pour une utilisation nomade de son poste.
Voir "Copier toutes les données utiles à une utilisation nomade".

¨   L'utilisateur fait des devis ou saisit des notes sur le poste nomade déconnecté du serveur.

¨   Une fois de retour, il transfert les nouvelles données de son poste à la base de données réseau.
Voir "Transfert des nouvelles données entre la base nomade et la base réseau".

Pour un poste qui n'est jamais connecté au serveur :

¨   Sur le serveur, l'utilisateur fait une sauvegarde de la base de données "réseau" sur un support amovible (disquette, cd-rom, IomégaZip, etc.) et la restaure sur le poste distant.
Voir le chapitre sur les sauvegardes et restaurations.

¨   L'utilisateur fait des devis ou saisit des notes sur le poste distant.

¨   Pour mettre à jour la base réseau avec les nouvelles données, l'utilisateur crée un fichier de synchronisation qu'il peut envoyer automatiquement par e-mail ou copier sur disquettes.
Voir "Créer un fichier de synchronisation".

¨   Les données de ce fichier peuvent ensuite être intégrées aux données du serveur.
Voir "Récupérer les données d'un fichier de synchronisation".
