Actions liées aux destinataires d'une note

Précédent   Suivant

Pour demander à une personne de faire quelque chose, vous pouvez lui envoyer un message en indiquant la tâche à effectuer dans le texte du message. Pour savoir si la tâche est faite, il faut que le destinataire vous contacte pour vous le signaler.

Cette façon de procéder n'est pas très efficace. Elle oblige le destinataire à envoyer un message de confirmation. De plus, elle ne permet pas d'avoir un suivi de toutes les actions que l'on a demandées. C'est pourquoi, les notes intègrent la notion d'Actions liées aux destinataires.

 

A chaque destinataire "interne", on peut définir une action que celui-ci devra effectuer. Le destinataire sera informé du message qui lui indiquera quelle action il doit faire. Le message est notifié au destinataire tant que l'action n'est pas faite.

Le principal intérêt de cette méthode, est que l'expéditeur peut avoir un suivi des actions qu'il a demandé. Il peut ainsi demander la liste des messages contenant des actions non effectuées.

 

Les actions que l'on peut lier aux notes doivent être paramétrées. Pour cela voir le paragraphe "Actions" dans le chapitre " Paramétrage de GS CRM".

La saisie d’une action à effectuer lors de la création du message, doit être faite dans l'écran de sélection des destinataires. Pour plus d'informations, voir le paragraphe "Choix des destinataires - Saisie classique".
