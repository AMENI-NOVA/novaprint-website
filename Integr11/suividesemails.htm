Suivi des e-mails

Précédent   Suivant

Vous disposez de deux outils pour le suivi des e-mails envoyés : « la confirmation de lecture » et « la preuve de dépôt ».

Confirmation de lecture
Demander une confirmation de lecture

Lors de la saisie de votre message (e-mail), vous avez la possibilité de demander une confirmation de lecture.

Vous pouvez ainsi recevoir un e-mail vous informant que le destinataire a bien lu la note. Dans ce cas, la date de lecture est affichée.

 

Remarque : Même si vous demandez une confirmation de lecture, rien ne garanti que vous allez la recevoir quand l’e-mail sera lu par le destinataire. Pour recevoir une confirmation de lecture, il faut :

1. Que le la messagerie du destinataire soit capable de lire les demandes de confirmations de lecture;

2. Que le destinataire accepte d’envoyer cette confirmation.

 

Pour demander une confirmation de lecture, vous devez le faire lors de la création du message. Pour cela, utilisez le menu « Suivi des e-mails », puis « Oui » dans la section « Confirmation de lecture ».

Il est possible de paramétrer un comportement par défaut (toujours envoyer, jamais ou demander). Pour cela utiliser le menu « Paramétrage des choix par défaut » (voir « Préférences d’utilisation : Suivi des e-mails »).

 

Un e-mail pour lequel vous avez demandé une confirmation de lecture est affiché avec l’icône suivante :

 

Lorsque vous recevez une confirmation de lecture :

¨   GS CRM fait le lien entre la confirmation de lecture et le message d’origine.
La zone « Note d’origine » affiche le message concerné par cette confirmation.

¨   GS CRM indique, dans le message d’origine, la date de lecture par le destinataire.

 

Réception d’une demande de confirmation de lecture

Lorsque que vous recevez un e-mail contenant une demande de confirmation de lecture, dès que le message à été lu, vous avez les choix suivants :

¨   GS CRM vous demande quoi faire :
XXX a souhaité être prévenu lorsque vous lirez le message ‘YYY’
Acceptez-vous d'envoyer une confirmation de lecture ?
Oui / Non

¨   Envoyer la confirmation de lecture automatiquement et sans intervention de votre part.

¨   Ignorer systématiquement.

Le comportement par défaut est défini dans les préférences d’utilisation de la messagerie (voir « Préférences d’utilisation : Suivi des e-mails » ).

 

La confirmation de lecture envoyée à l’auteur de l’e-mail est comme suit :

Votre message

 

     A : xxx@yyy.fr

     CC : zzz@www.fr

     Objet : Ce mail contient une confirmation de lecture

     Date : 28/03/2011 11:25:38

 

 a été lu le 28/03/2011 11:26:05.

 

Note : Cette confirmation indique seulement que le message a été affiché sur l'ordinateur du destinataire.

Il n'y a aucune garantie que le destinataire ait lu ou compris le contenu du message.

 

Preuve de dépôt

Une preuve de dépôt (également appelé accusé réception) permet d'obtenir la confirmation que les messages ont bien été déposés dans le compte de messagerie des destinataires.

Si vous demandez une preuve de dépôt, vous recevrez immédiatement un e-mail vous indiquant si le message à bien été remit aux destinataires. Attention, cela ne signifie pas que le message à été lu. Pour cela il faut utiliser la demande de confirmation de lecture.

 

Remarque : Même si vous demandez une preuve de dépôt, rien ne garanti que vous allez la recevoir. Cela dépend de serveur Smtp utilisé pour envoyé l’e-mail. Tous les serveurs de messagerie ne supportent pas cette fonctionnalité.

 

Pour demander une preuve de dépôt, vous devez le faire lors de la création du message. Pour cela, utilisez le menu « Suivi des e-mails », puis « Oui » dans la section « Preuve de dépôt ».

Il est possible de paramétrer un comportement par défaut (toujours envoyer, jamais ou demander). Pour cela utiliser le menu « Paramétrage des choix par défaut » (voir « Préférences d’utilisation : Suivi des e-mails »).
